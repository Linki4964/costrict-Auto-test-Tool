# 模式规定
- 每一次前端测试都要开启有头模式
- 编码格式全局采用GBK编码格式
- admin为超级管理员禁止删除

# 强制性约束命令
- 严格按顺序执行 Step 1 到 Step 4，禁止跳过任何步骤。
- 严禁进行主观简化。即使你认为可以合并步骤，也必须分步输出。
- 每个步骤的输出是下一个步骤的唯一输入，未检测到上一步输出严禁启动下一步

# 文件命名规范
## 保存路径规范
每次开启新任务都需要创建一个新的文件
文件夹命名格式规范
```
[时间]+[任务名称]
```
每次任务生成的文件都必须要新建一个独立文件夹里保存：
```
/工作区/[新文件夹]
```
## 生成文件规范
所有生成的文件都必须按照一下规范进行生成：
```
[日期]_[模块/功能]_[测试内容]_[版本].[文件后缀]
```

禁止生成任何没有被允许的文件夹，禁止自行命名创造新的文件夹
